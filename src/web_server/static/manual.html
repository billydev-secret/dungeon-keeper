1 Functional Blocks at a Glance

DungeonKeeper is an all-in-one server management bot. It handles moderation, ticketing, XP tracking, voice channels, analytics, and more — all through Discord slash commands.

The bot is split into ~30 cogs (functional blocks). This table is the bird's-eye view — each row links down to the section that documents the commands. New admins should jump straight to §16 Setup & Permissions first.

2 Moderation Core

Jail & Release

The jail system moves a member to a private channel, strips all their roles, and logs everything. Jail transcripts are saved automatically for later review.

/jail user:@user duration:24h reason:Repeated rule 3 violations Mod Places the member in a private jail channel and removes all their roles. The bot DMs them with the reason.
duration — accepts human durations: 30m, 2h, 7d, 1w, or leave blank for indefinite.
reason — logged to audit trail and included in DM.
/unjail user:@user reason:... Mod Releases the member, restores their previous roles, and saves a full transcript of the jail conversation.
ℹ️
Automatic release

Timed jails expire automatically — you don't need to manually unjail after the duration. The bot checks every 30 seconds and releases with a logged reason.

Member Profile

/modinfo user:@user Mod Full mod profile for a member — current jail status, all warnings, open tickets, join date, and account age.

Bulk Delete (Purge)

/purge count:50 after:14:30 Mod — requires Manage Messages Bulk-deletes messages. Specify a count, a cutoff time (HH:MM in server timezone), or both. Pinned messages are never deleted.
count — 1–1000 messages.
after — only delete messages sent after this time today (e.g. 14:30).
💡
Pro Tip: Right-click context menus

Faster than typing slash commands when you're already looking at the offending user or message. Right-click reveals these actions:

Right-click targetActionWhat it does
User Jail User Opens a modal to quickly jail the member without typing a slash command
Message Open Ticket About This Message Opens a ticket pre-linked to the offending message
Message Add to Todo Adds the message to the server todo list with an optional note (mod-only)
Message Steal Emoji Extracts any custom emojis from the message and lets you add one to a server you manage. Also available as /steal_emoji url:<url> name:<name> for grabbing emojis directly from a URL.
Message Quote Generates a styled quote-card image from the message and posts it as a reply

3 Tickets, Policies & Warnings

The escalation toolkit. Tickets create private channels between the member and the mod team (everything logged). Policies let mods propose and vote on rules. Warnings build a paper trail per member.

⚠️
Use tickets, not DMs

Handling moderation issues in DMs leaves no record. Open a ticket so the full conversation is logged, searchable, and reviewable by the whole team.

💡
Pro Tip: Warn / Jail / Ticket — pick the right escalation

Warn when the rule violation is clear-cut and you want a paper trail without a conversation. Ticket when the situation needs context-gathering, the member has questions, or you'd otherwise be DMing them. Jail when the member needs to stop interacting now — it strips roles, cuts visibility, and saves a transcript automatically. A jail can absorb a ticket conversation; a warning can't.

Warnings

/warn user:@user reason:... Mod Issues a warning to a member. Warnings are private — the member is not notified. Stored in the audit log. Admins are pinged when a threshold is reached.
/warnings user:@user Mod Lists all active and revoked warnings for a member, with dates, reasons, and which mod issued each one.
ℹ️
Revoking warnings

Run /warnings @user to list warning IDs, then /revokewarn user:@user warning_id:<N> reason:... to revoke one. Revoked warnings stay in history (they don't disappear), they just no longer count toward thresholds.

Ticket Commands

/ticket panel Mod Re-posts the persistent "Open a Ticket" panel button in the configured ticket channel. Use after the panel scrolls off or after a bot restart.
/ticket open Everyone Opens a private ticket channel visible to the member and all mods. Members can describe their issue in the opening modal.
/ticket claim Mod Assigns the ticket to you. You'll receive DM pings for new messages in that ticket until it's closed.
/ticket escalate reason:... Mod Brings admins into the ticket with a ping. Use when the issue is above your authority level.
/ticket close reason:... Mod Locks the channel and posts Reopen / Delete buttons. The channel remains visible for review until deleted.
/ticket reopen Mod Re-opens a closed ticket (also available via the Reopen button posted on close). Restores write access for the ticket member.
/ticket delete Mod Permanently deletes the ticket channel. A full transcript is saved to the database first.
/pull user:@user    /remove user:@user Mod Add or remove a member from the current ticket/jail channel. Useful for bringing in witnesses or additional mods.

Policy Voting

The policy system lets mods propose and vote on server rules democratically.

/policy open title:... description:... Mod Creates a policy proposal with Yes / No / Abstain voting buttons for the mod team.
/policy list    /policy close    /policy vote Mod List all policies, finalize voting on a proposal, or trigger a vote on an existing proposal.

4 Analytics & Watch List

💡
Pro Tip: Catch problems before they escalate

The analytics tools let you identify who's going quiet, who's stirring conflict, and who's at-risk of churning — before it becomes a problem. The full report set lives in the web dashboard (open it here); check the Drop-off and Chilling-effect reports weekly to get ahead of server dynamics instead of reacting to them.

Watch List

Silently monitor a member's public messages, forwarded to your DMs.

/watch add user:@user Mod Starts forwarding the member's public messages to your DMs. Completely invisible to the member.
/watch remove user:@user    /watch list Mod Stop watching a member, or see everyone you're currently monitoring.

Activity & Engagement Analytics

/activity resolution:day member:@user channel:#channel Mod — requires Manage Guild Bar chart of message volume or XP over time. Scope to a specific member, channel, or the whole server.
resolution — hour / day / week / month / hour_of_day / day_of_week  |  mode — messages / xp.
ℹ️
Drop-off, Session Burst, Burst Ranking, Chilling Effect

These engagement reports moved to the web dashboard. /activity stays in Discord for quick spot-checks; everything else is interactive on the dashboard with filters, sorting, and exports.

6 Community & XP

XP System

Members earn XP by posting messages, replying, reacting to images, and spending time in voice channels. XP level-ups automatically grant configured roles and post announcements.

/xp_leaderboards timescale:week Everyone Shows the top XP earners and your own rank, with a breakdown by source (text / reply / voice / image-react / grant).
timescale — hour / day / week / month / year / alltime.
/xp_give member:@user Mod or allowlisted members Manually awards 20 XP to a member. Use to recognize contributions that the bot wouldn't automatically capture (e.g. helping in voice, organizing events).

Role Grants

/grant role:denizen member:@user Configured per role Grants a configured community role to a member. Role options and who can grant them are set in /config roles. The action is logged and optionally announced.
/report promotion_review Mod — Manage Messages Lists members at Level 5 or above who haven't yet been granted the NSFW or next-tier role. Flags pruned members. Use during weekly check-ins.
/quality_leave add member:@user days:30 Mod Puts a member on Leave of Absence so their inactivity doesn't count against quality scoring. Use for members who announced a temporary break.
/quality_leave remove member:@user    /quality_leave list Mod End a LOA or see all members currently on leave.

Starboard

/starboard channel:#best-of   /starboard threshold:5   /starboard toggle   /starboard status Mod Configure and enable the starboard. When a message receives enough star reactions, it's reposted to the starboard channel as a highlight.
/starboard emoji ⭐ Mod Change which reaction emoji counts toward the threshold. Defaults to ⭐. Accepts unicode or a custom emoji from this server.
/starboard exclude #channel   /starboard unexclude #channel Mod Channels in the exclusion list never have their messages mirrored to the starboard, even if they get enough stars.

9 Confessions

Anonymous Confessions

The confessions system lets members post anonymously to a configured channel. Each confessor is assigned a colored emoji (e.g. 🔴🟢🔵) so mods can identify repeat posters without exposing identity to other members.

/confess Everyone Opens an anonymous submission modal (up to 4,000 characters). The confession is posted to the confessions channel with a colored emoji identifier visible only to mods.
ℹ️
Mod capabilities in confessions

Mods can see the emoji→member mapping in the mod log channel, block a specific member from confessing (without revealing their identity), and enable panic mode to temporarily suspend all new confessions.

10 DM Permissions

Members declare a DM "mode" via Discord roles (Open, Ask, or Closed). The bot enforces requests through a button panel and modal, automatically deduping conflicting role assignments.

DM Commands

/dm_help Everyone Show an overview embed explaining the three DM modes, what each means, and the commands available.
/dm_set_mode mode:[open|ask|closed] Everyone Set your own DM mode. open = anyone may DM you. ask = approval required (default). closed = no DM requests accepted.
/dm_request_panel_refresh Mod — Manage Channels Re-post the persistent DM request button panel so it's at the bottom of the configured channel. Background bumping is automatic; this is a manual override.
/dm_revoke user:@member Everyone Remove an existing mutual DM connection. Both parties are DM'd a notice and the action is logged to the audit channel.
/dm_status user:@member Everyone Check whether mutual DM permission currently exists between you and another member.

11 Wellness System

The wellness system helps members maintain healthy Discord habits. Members opt in for themselves; admins can also create caps, blackouts, and exemptions on a member's behalf.

Member Commands

/wellness setup Everyone Opens the enrollment wizard. Member selects their timezone and enforcement level.
Enforcement modes:
Gentle — friendly DM reminders when activity spikes.
Cooldown — short posting breaks when threshold exceeded.
Slow Mode — Discord per-user slow mode applied automatically.
Gradual — escalates from gentle → cooldown → slow mode.
/wellness away on text:<custom>    /wellness away off Everyone Enables an away auto-reply. When someone @mentions an away member, the bot replies automatically with a default or custom message (max 500 chars).

Admin Commands

The /wellness-admin group is admin-only. Use it to create infrastructure, set server defaults, and override individual users' caps / blackouts / settings on their behalf.

💡
Pro Tip: When to admin-override vs let the member self-manage

Wellness is opt-in by design — most members should run /wellness setup themselves. Use admin overrides only when (a) the member has explicitly asked you to set something up they can't reach (e.g. accessibility, partnership setup), or (b) they're already enrolled and visibly struggling — in which case /wellness-admin dashboard first to see who's in slow mode, then /wellness-admin cap add or blackout add on their behalf. Don't impose caps on members who never opted in; that's a moderation problem, not a wellness one.

CommandWhat it does
/wellness-admin setup [role_name] [channel_name] [crisis_resource_url]First-time setup. Creates the wellness role and a private wellness channel.
/wellness-admin defaults [default_enforcement] [crisis_resource_url]Set server-wide enforcement default (gentle / cooldown / slow_mode / gradual) and crisis-resource URL.
/wellness-admin dashboardServer-wide stats embed: participants, slow-mode count, active caps & blackouts, partnerships, top streaks.
/wellness-admin exempt add / remove / listMark channels exempt from cap counting (e.g. mod-only channels). Server-wide.
/wellness-admin cap add user:@m label scope:[global|channel|category|voice] window:[hourly|daily|weekly] limit:<n>Create a message cap on a member's behalf. channel/category scopes require the corresponding argument. exclude_exempt defaults true.
/wellness-admin cap edit user:@m label new_limit:<n>Change a cap's limit. Label autocompletes from that user's caps.
/wellness-admin cap remove user:@m labelRemove a cap.
/wellness-admin blackout add user:@m name start_time:HH:MM end_time:HH:MM days:<list>Daily slow-mode blackout window. days: all / weekdays / weekends / mon,wed,fri.
/wellness-admin blackout remove user:@m nameDelete a blackout. Name autocompletes.

13 Network Analytics

Force-directed graph and matrix views of who interacts with whom. Useful for spotting cliques, isolated members, and invite trees. The first time you run these, also run an /interaction_scan backfill to populate history.

Connection Graphs

/connection_web [member:@user] [timescale:hour|day|week|month|all] [min_pct:1-100=5] [layers:1-5=2] [limit:5-60=40] [spread:0.5-5.0=1.0] [max_per_node:0-20=3] Mod Render the reply / mention network as a force-directed graph. With a member, focuses on them and expands outward by layers. min_pct hides low-volume edges; max_per_node caps the fan-out per person.
/interaction_heatmap [timescale:hour|day|week|month|all=all] [min_pct:1-100=5] [limit:5-60=30] Mod Adjacency-matrix heatmap — useful when you want to see clusters rather than a graph layout.
/invite_web [member:@user] [spread:0.5-5.0=1.0] Mod Network graph of who invited whom. Without a member, shows the full forest; with a member, shows their invite subtree.

Backfill Scans

/interaction_scan [days:0-3650=0] [reset:bool=false] [channel:#chan] Mod Backfill the interaction graph from message history. days=0 means everything available. reset=true wipes existing interaction data first. Limit to a single channel for incremental backfills.
/member_events_scan [days:0-3650=0] Mod Backfill join / leave history from the configured leave-log channel. Required before invite-tree analysis is meaningful on older servers.

14 Cleanup & Tagging

Auto-Delete

Drop-in message cleanup. Schedule a channel to auto-delete posts older than a given age, on a recurring or one-shot basis. Pinned messages are never deleted.

/auto_delete del_age:30d run:once Mod Delete messages in this channel older than del_age. run accepts once (run now), off (cancel any schedule), or an interval (1d, 12h, 30m) to enable a recurring schedule.
del_age / run — accept human durations: 30d, 2h, 15m, 1h30m.
/auto_delete_configs Mod Open an interactive panel listing every auto-delete schedule in this server with Edit / Remove / Clean-up-missing buttons. Editing a rule lets you change the age and interval, or set the interval to off to delete the rule.

Inactivity Prune

Auto-removes a configured role from members who go inactive for X days. Runs daily at midnight UTC.

/inactivity_prune status Mod — Manage Server Show the configured role, threshold (days), and current exemption list.
/inactivity_prune exempt member:@user    /inactivity_prune unexempt member:@user Mod Add or remove a member's exemption from the daily prune.
/inactivity_prune run Mod Trigger a prune pass now instead of waiting for the daily schedule. The prune rule itself is set in /config prune.

Gender Tagging

Tag members by gender to enable analytics breakdowns (e.g. NSFW-by-gender chart on the dashboard).

/gender set member:@user gender:[Male|Female|Non-binary] Mod Set or change a member's gender tag.
/gender check member:@user    /gender classify Mod View one member's tag, or step through every unclassified member one at a time with M / F / NB / Skip / Done buttons.

15 Special Events

April Fools' Nickname Shuffle

One-day prank: scrambles the nicknames of recently-active members in a 1:1 derangement. Members can opt out, mods can repair if it goes sideways.

/foolsday action:[shuffle|restore] Mod — Manage Server Shuffle nicknames among active members, or restore originals from the saved table.
/foolsday_exclude [user:@m]    /foolsday_join [user:@m] Self only / Mod for others Opt out of the shuffle, or join one already in progress (swaps your name with a random current participant). Without arguments, applies to yourself.
/foolsday_include user:@m    /foolsday_exclusions Mod Re-include a member previously excluded, or list everyone who's opted out.
/foolsday_samename [name:<text>] Mod Set every shuffled member's nickname to one shared name (custom or randomly picked from the saved pool).
/foolsday_repair Mod Recovery tool: walk the audit log, identify the bot's first nickname change per member, and restore originals. Use when the saved-name table is corrupted.

Veil Image Game

NSFW image-guessing game. Submit a cropped picture; opted-in members try to identify whose body it is. The full image is revealed (spoilered) on the first correct guess.

/veil submit image:<attachment> Configured Veil role (or Everyone if unset) Submit an NSFW image. The bot validates dimensions, runs the crop pipeline, and shows you a preview with Re-roll / Post buttons. Posting publishes a public game message to the configured Veil channel.
/veil setup channel:#nsfw [role:@role] Admin — Manage Server First-time setup. Set the Veil channel and (optionally) the role required to submit. Without a role, every NSFW-permitted member can submit.

Whisper Anonymous Messages

Anonymous message-guessing game. Send a secret message to an opted-in member via DM; they can try to guess who sent it, share it to the server feed, hide it for later, or reply anonymously.

/whisper send target:@user message:Your secret message Opted-in members (hold the configured Whisper role) Delivers an anonymous message to another opted-in member via DM. Only opted-in members appear in autocomplete. Cooldown: 30 seconds between sends; max 5 whispers to the same person per hour.
/whisper optin Everyone Opt in to send and receive anonymous whispers. Grants the configured Whisper role after a confirmation prompt.
/whisper optout Everyone Opt out of whispers. You'll no longer appear in others' autocomplete. Existing whispers are preserved.
/whisper forget-me Everyone Permanently deletes all whispers you sent or received in this server. Cannot be undone.
ℹ️
Inbox actions

When a whisper arrives in your DM you'll see buttons to Guess the sender (limited attempts), Share it to the public feed, Hide it from your main inbox, Reply anonymously, or Report it to moderators. Identities are always logged for moderators even though senders appear anonymous to recipients.

16 Setup & Permissions

Permission Levels

Most commands require one of these roles or Discord permissions:

LevelHow it's grantedWhat it unlocks
Everyone All server members Opening tickets, XP leaderboards, voice channel controls (own channel), wellness opt-in, confessing
Mod Has the configured mod role, or Manage Guild permission Jail, warnings, tickets management, watch list, analytics, purge, XP awards
Admin Has the configured admin role, or Discord Administrator permission User data deletion, voice admin, policy voting finalization
Bot Owner Configured at bot startup Reloading cogs, Spotify authorization

First-Time Setup (Admin)

  1. Run /setup — Phase 1 creates all required bot channels and categories (mod-log, join-leave-log, ticket panel, etc.); Phase 2 walks you through a 6-step wizard for mod/admin roles and jail/ticket categories.
  2. Run /config global — set your mod role IDs, admin role IDs, mod notification channel, and server timezone.
  3. Run /config roles — configure which roles can be granted with /grant.
  4. Run /config xp — set the Level 5 role, XP announcement channels, and excluded channels.
  5. Run /config welcome — customize join/leave messages.
  6. Optionally run /voice-admin set-hub to enable personal voice channels.
  7. Optionally run /starboard channel to enable the starboard.
