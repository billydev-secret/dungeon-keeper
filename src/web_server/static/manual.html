    DungeonKeeper

    User & Moderator Guide
    👥 For members, mods & admins 🤖 Discord Server Bot + Web Dashboard

    1 Getting Started

    DungeonKeeper is an all-in-one Discord server companion: moderation, support tickets, XP, personal voice channels, music, anonymous games, wellness tools — plus a web dashboard for settings and reports. You don't need to learn all of it at once. Start with the track that matches you.

    Discord vs. the dashboard

    The bot has two halves, and knowing which is which saves a lot of hunting:

    • Discord slash commands are for doing things — jail a member, open a ticket, play a song, start a game.
    • The web dashboard is for settings and reports — every persistent option lives in a dashboard panel, and so do all the analytics. Sign in with your Discord account; what you can see depends on your server roles.

    Throughout this guide, command permissions are shown as badges: Everyone any member, Mod configured mod role, Admin configured admin role, Bot Owner the bot's operator. Details in Permission Levels.

    I'm a member — quick tour

    • Find any command: type /help in Discord for a browsable list of everything available to you.
    • Hang out: join the voice hub channel to get your own personal voice channel, or queue music with /play (Music).
    • Play: /games help lists every party game — see Games Night, Guess Who, and Whisper.
    • Express yourself: post an anonymous confession with /confess, set a /bio, register your /birthday set.
    • Set boundaries: control who can DM you (DM Permissions) and opt into healthy-usage nudges (Wellness).
    • Own your data: /delete_me erases your data — see Data Erasure.

    I'm setting up the server — quick start

    1. Run /setup in Discord — it creates the bot's channels and walks you through mod/admin roles.
    2. Open the web dashboard, sign in with Discord, and work through the panels in the First-Time Setup checklist.
    3. Skim the Feature Map below and enable only what your community needs — every feature is independent and most are off until configured.

    Finding things in this guide

    • Search — use the search box in the sidebar (or in the dashboard's Help panels) to search the entire guide by command or keyword.
    • Feature Map — one-line summary of every feature, linked to its section.

    Ask Billy-bot (AI helper)

    If you'd rather just ask, Billy-bot is an AI helper that answers "how do I…" questions in plain language, grounded in this guide:

    • In Discord — run /ask with your question (e.g. /ask how do I start a game?). The reply is private (only you see it).
    • On the dashboard — use the Ask Billy-bot box at the top of any Help page.

    Billy-bot only answers from what it's been given, so it won't invent commands — if it's not sure, it'll say so and point you here or to a moderator.

    Admins get more from the Config → Billy-bot panel: it picks the model — separately for regular members (Haiku by default) and for mods/admins (Sonnet by default, since staff questions are the ones that look up and change settings) — and can turn on live server context (off by default) so Billy-bot answers from this server's channel topics, pins, announcements, dashboard docs, and saved settings — always scoped to what the asker can see, never NSFW. Admins can also ask it "what am I missing?" — it reviews which features this server hasn't set up (or has set up but left switched off) and suggests the cheapest wins first. And an admin can ask it to change a setting or set a feature up from scratch; it never applies anything itself, but returns permission-checked Apply buttons to confirm, one per setting. That includes your role grants (NSFW, Denizen, Veteran and any custom ones) — you can ask it to point a grant at a different role, change where grants are logged, or reword the grant message, though creating a brand-new grant stays a dashboard job. Settings that hand out access or moderation authority — role grants, the jailed role, who can mark Q&A answers, who can whisper, the unverified role — need a full server administrator, not just Manage Server, and that's re-checked when the button is pressed. A short list is off-limits entirely, whoever asks: your admin and mod role IDs, and the message-storage privacy level. Richer feature settings (economy prices, quests, and the like) stay on the dashboard. Full details in Configuration.

    3 Games Night

    A suite of party games, duels, and group elimination games for hangouts and game nights. Everything is member-facing — anyone can start a game in an allowed channel. Only one game runs per channel at a time. Type /games help in Discord for the in-app version of this list.

    💰
    Coin wagers

    All six duel and group games take an optional wager: amount — every player antes the same, and the winner takes the pot. Some servers set a small house cut on won pots; when one applies, the payout message says exactly how much the house kept. Refunds are never cut. In a 1-v-1 challenge nothing is charged until the target accepts (declining or letting it time out costs nothing); in a lobby game you pay as you join, and leaving the lobby refunds you. If a game is abandoned, voided, cancelled, or ends with no winner, every stake is returned — and if you leave the server mid-game your stake comes back automatically. Wagers need the economy enabled and use your /bank wallet balance. A wager replaces the usual nickname stake: the pot is what's on the line, so the winner doesn't get the rename button. Nicknames are only at stake in the default game — no wager:, no custom stakes:.

    4 Photo Challenge

    A standalone scheduled photo feature (it used to live in the Games suite). On its configured schedule the bot posts a photo-prompt card to a dedicated Photo Challenge channel, optionally pinging a role. Members answer by posting their photos in that same channel — no commands involved.

    💡
    Posting is the reward

    When the Economy is running, simply posting an image in the Photo Challenge channel pays out a flat participation award — no reactions or replies needed. If there's also an active photo_post quest, its reward stacks on top of the participation award. Both are capped at once per member per day, no matter how many photos you post.

    ℹ️
    Configure via the web dashboard

    Admins (or the game-host role) set the channel, ping role, recurring schedule, and enabled toggle on the Photo Challenge → Setup & Schedule panel. The flat participation award is a faucet rate on Economy → Income Sources ("Photo Challenge post"); the optional stacking bonus is a photo_post quest in Economy → Quests. The prompt bank and AI prompt studio live under Photo Challenge → Prompts & AI. The old /games play photo command is retired.

    7 Confessions

    Anonymous Confessions

    The confessions system lets members post anonymously to a configured channel. Each confessor is assigned a colored emoji (e.g. 🔴🟢🔵) so mods can identify repeat posters without exposing identity to other members.

    /confess Everyone Opens an anonymous submission modal (up to 4,000 characters). The confession is posted to the confessions channel with a colored emoji identifier visible only to mods.
    ℹ️
    Mod capabilities in confessions

    Mods can see the emoji→member mapping in the mod log channel, block a specific member from confessing (without revealing their identity), and enable panic mode to temporarily suspend all new confessions.

    8 Pen Pals

    Pen Pals quietly matches members into private one-on-one conversations. Joining matches you straight away if someone eligible is already waiting — otherwise you sit in the pool and are matched the moment the next person joins. Either way the bot spins up a private two-person channel, seeds it with a conversation-starter question, and tears it down again after a configurable session length (default 24 hours). Nobody is ever in two pen pal chats at once, and after a chat ends there's a re-match cooldown (set on the dashboard) before you can be paired again. If there's someone you'd rather not be matched with, /penpals block keeps the two of you apart — privately, and one block is enough.

    /penpals blockEveryoneOpen a private panel to pick members Pen Pals should never match you with (or unblock them) — one block is enough, the other person is never told, and it doesn't end a chat you're already in

    10 Economy & Perk Shop

    A per-server currency (branded per guild — name, emoji, and wallet name are all configurable) earned by being active and spent on weekly-renewing perks. There's no grinding loop of earn commands: currency flows from the activity you're already doing — logging in, hanging out in voice, playing games, answering the question of the day, and completing quests. The economy is off until an admin enables it and sets a bank channel.

    Earning

    • Daily login — your first counted message or qualifying voice session of the day pays a small login reward (voice pays more than text). Consecutive days build a streak bonus, one missed day per week is silently forgiven, and milestones (day 7, 30, 100…) pay extra.
    • Quests — daily / weekly / monthly quests drawn onto your personal board, plus server-wide community goals and event quests.
      • Many complete automatically when you do the thing (send a trigger phrase, play a game, post a photo, answer a chat-revive prompt, host a voice hangout, set a bio…); others you claim from /bank quests, and a few need a staff sign-off before paying.
      • Some quests travel in pairs — draw one half (say, submit a Guess Who round) and its partner (play a round) lands with it.
      • Getting-started nudges — one-time setup tasks like filling out your bio or setting your birthday — appear among your dailies and quietly drop off once done, so they only ever show to people who haven't.
    • Community weeklies — every other week the server gets a shared goal ("send 3,000 messages together") sized from recent activity. Everyone's actions count automatically, milestones at 40/70/100% each pay every active member, and top contributors earn a bonus — so even a partial week pays out.
    • Board extras — perks on your quest board:
      • One free 🎲 reroll per day on /bank quests swaps out a quest that doesn't fit; buy a few more that day if you want another (the select shows the price).
      • Clearing your whole daily or weekly board pays a set bonus.
      • Each week one ⚡ spotlight activity pays double (announced when the new week's quests go up).
    • Games & QOTD — playing a party game or duel pays participation (winning pays more; the results card shows the amounts). Replying to the question of the day pays once per QOTD — use Discord's reply on the question itself; just talking in the channel doesn't count, and yesterday's question stops paying at the day roll.
    • Coin drops — where enabled, the bot occasionally drops a pouch of coins into an active chat channel. First to hit the 🪙 Claim button takes the whole pouch; the message updates to show who won, and anyone slower is told privately. Unclaimed pouches expire, and drops only land where there's conversation.
    • Server boosters earn 1.5× on everything the server pays out (member-to-member transfers are never multiplied).

    The Ledger

    The dashboard panel's Ledger tab is a separate thing from the alert queue, and it works differently on purpose. Where the queue holds the AI's opinion that a message might break a rule, the ledger records only that a specific, narrowly-defined thing was said. It makes no judgment, sends no alert, and has no Confirm / False-positive buttons — there is nothing to agree or disagree with. It exists so that if you are already reviewing someone, the relevant dates are on hand rather than reconstructed from memory.

    Two entries are recorded:

    • DM consent — someone says the consent bot is broken or that they don't know how to use it, and expresses intent to DM a particular person. Asking publicly before DMing is encouraged and never recorded; what lands here is the claim that the bot can't be used.
    • Cross-platform — someone shows they have already looked at another member's content on Reddit, Instagram, OnlyFans, Snapchat, Telegram or TikTok. Asking someone for their handle, or recognizing them from another platform, is ordinary and is not recorded — only demonstrating you went and looked. If the person raised that platform themselves in the last month, nothing is recorded.

    Both are deliberately rare — measured across fifteen months of server history they fire about once a month between them. If the ledger starts filling up, something has been widened and should be looked at.

    ℹ️
    The ledger runs even when the AI doesn't

    Unlike the alert queue, the ledger needs no AI model — it is plain text matching, so it keeps working if the AI service is unavailable. It still requires Rules Watch to be enabled.

    /rules-watch digest Mod Post a summary of unlabeled digest-tier events for batch review (up to 25 pending events per digest).
    /rules-watch label event_id:<n> verdict:[violation|fp] Mod Manually label a queued event as a violation or false positive. Labels tune the signal over time.
    event_id — numeric ID shown in the digest embed.
    verdict — violation (confirmed real) or fp (false positive).
    corrected_rule — optional: the correct rule number, if the guard flagged a real violation but matched the wrong rule.
    /rules-watch stats Mod View signal firing rates, label counts, and the false-positive rate.

    Inactive Members

    A softer sibling of the jail system for idle members: instead of a per-user cell, everyone held inactive shares one channel. Roles are snapshotted and stripped, the member gets an @Inactive role that can only see that channel, and a persistent panel there lets them open a ticket to be reactivated. Members enter manually or via a sweep; the only way out is /inactive release.

    /inactive panel channel:#inactive Admin — Manage Guild One-time setup: sets the shared inactive channel, creates/wires the @Inactive role, and posts the info panel with an "Open Ticket" button. Required before mark or sweep will run.

    /inactive sweep [apply:true|false] Admin — Manage Guild Defaults to a dry run: lists members idle past the threshold without touching anyone. Pass apply:true to actually move each listed candidate through the mark flow.
    ℹ️

    Greeting Watch

    Catches "good morning" / "hello" messages in your main chat that go unanswered, so nobody who says hi to the room falls through the cracks. When a short greeting in a watched channel isn't replied to or @mentioned within the configured window (default 10 minutes), the bot DMs the chosen members — greeters or admins — with a jump link. You can subscribe one member or several. Only IDs and timestamps are stored, never message text. There are no slash commands.

    ℹ️
    Configure via the web dashboard

    The Greeting Watch panel has the enable toggle, the watched channel list, who gets the DM (one or more members), and the unanswered-window length.

    Intake Cards

    Turns your welcome procedure into a live checklist. When someone joins, the bot posts an intake card to greeter chat — pinging the greeter role instead of the old bare @here line — showing a progress bar, the member's account age, who invited them, and the procedure as ✅/⬜ steps. The card is a tracker, not a control panel: welcomers keep greeting, asking the question lists, and running /grant exactly as before, and the card ticks itself — a greeter mentioning the newcomer ticks Greeted, the verification role lifting ticks Verified, and gaining a configured role (via /grant or a manual add) ticks the matching role step. The question phases are buttons welcomers tick by hand. Nothing anyone says is ever recorded — only that steps happened, by whom, and when.

    To finish an intake, the welcomer posts the ending welcome message containing the configured completion code and @mentioning the newcomer — any channel works. The card completes, the poster is recorded as the welcomer, and any unticked steps are marked skipped (visible in analytics, never blocking). Cards only ever close on completion, a mod's Dismiss, or the member leaving or being banned — so the open cards are your intake queue. A card with no progress for the configured window gets one greeter-role nudge reply, never more.

    The panel also manages a procedure reference channel (your #welcome-procedure): the text sections and question lists live in a block editor on the dashboard and the bot keeps the channel in sync — question lists post one message per question, so a greeter taps Copy Text on exactly the question they need. Rewording a question edits the same message in place (links keep working); a one-time Import button seeds the editor from the channel's existing content. The bot never touches messages humans posted there.

    ℹ️
    Configure via the web dashboard

    The Intake Cards panel has the enable toggle, card channel, completion code, stale-nudge window, the step-list editor (steps are snapshotted onto each card when it posts, so edits never disturb in-flight intakes), and the Procedure Reference block editor with its Import button. The Intake Queue report shows the open queue, time-to-complete, per-welcomer counts, and which steps get skipped.

    Chat Revive ("Ember")

    Notices when a channel has gone unusually quiet for that channel, at that time of day, and gently restarts conversation by posting a question card from a curated bank — optionally tagging an opt-in chat-revive role. It's not a timer bot: it learns each channel's rhythm, never talks over an active room, never posts twice in a row, and strict frequency protections (per-channel rest period, guild daily budget, quiet hours, at most one ping per channel per day) keep it rare enough to feel like a treat. Spicier questions only ever surface in age-restricted channels. Members join or leave the ping role through the server's usual self-role flow; there are no slash commands.

    ℹ️
    Configure via the web dashboard

    The Chat Revive panel covers eligible channels and their question flavors, the ping role, quiet hours and budgets, the question bank (add, bulk-import, retire), a "would it fire right now?" explainer per channel, a manual Fire button, posting the opt-in button, and a scoreboard of which questions actually spark conversation.

    Hidden Channels

    Hide a channel from everyone and later restore it exactly as it was. Hiding snapshots the channel's permission overwrites and placement, then parks it under a "Hidden Channels" category with @everyone denied View Channel (members with Administrator still see it — Discord always exempts them). Restoring moves it home and reinstates the original overwrites. Hides last until restored — there's no scheduled auto-restore.

    /hidden hide channel:    /hidden restore channel:    /hidden list Admin Hide a text / voice / stage / forum channel (categories can't be hidden), bring one back, or list what's currently hidden and who hid it. The bot needs Manage Channels and Manage Roles. No dashboard panel — these three commands are the entire interface.

    23 Content & Comms

    Bot-maintained content and outbound messaging: self-assign role menus, dashboard-authored documents posted into channels, and scheduled announcements. All three are built on the dashboard and post persistent messages that keep working across bot restarts.

    Role Menus

    Self-assign role menus, built entirely on the dashboard and posted into a channel as a persistent message with buttons or a dropdown. Members click to manage their own roles — the reply is only ever visible to them — and the message keeps working across bot restarts. Each menu has one of five modes:

    • Toggle — click to get the role, click again to drop it.
    • Unique — only ever one role from the menu at a time (picking a new one swaps it).
    • Verify — gain-only, for verification gates.
    • Drop — remove-only, an opt-out station.
    • Binding — the first pick is permanent, one ever per member.

    There are no slash commands — members just click the posted message.

    ℹ️
    Configure via the web dashboard

    The Role Menus panel is where you build a menu (roles, labels, emoji, mode, button vs. dropdown), pick its channel, and post or re-sync the live message. For a quick role button attached to a one-off post, see Announcements instead.

    Docs

    Author documents on the dashboard and post them into channels as bot-maintained messages — rules pages, guides, FAQ posts. Edit the doc on the dashboard and re-sync; the posted copies update in place.

    /docs post    /docs sync    /docs unpost    /docs list Mod — Manage Guild post puts a doc into a channel (and keeps it synced), sync re-renders a doc everywhere it's posted (leave the doc blank to sync all), unpost removes it from a channel, and list shows every doc and where it's posted. The documents themselves are written and managed on the dashboard's Docs panel.

    Announcements

    Queue server announcements on the dashboard, preview them, and set a post time — the bot posts them for you. Each announcement is a rich embed (title, markdown body, optional image, accent color that defaults to your server branding) plus an optional plain-text line shown above the embed. Mentions only ping from that plain-text line, and only when you explicitly pick them: no mentions, one role, or @everyone — nothing pings by accident. Times are server-local (the timezone offset from the Global config panel).

    ℹ️
    Manage via the web dashboard

    The Announcements panel (admin-only) has the queue, a live Discord-style preview while you type, and a Sent history. Save without a time to keep a draft; add a date and time to schedule it. Post Now sends within a minute. Sent items keep an "Open in Discord" jump link and can be cloned back into a fresh draft. If the bot was offline at post time it still posts up to 2 hours late; past that the item is marked as missed instead of posting stale news — clone and reschedule it.

    Role buttons

    An announcement can carry up to five role buttons. A member clicks one to give themselves that role and clicks again to drop it — the reply is only ever visible to them. Handy for opt-in pings, event signups, and notification roles: post the announcement once and the button keeps working from then on, including after the bot restarts and even if you later delete the announcement from the queue.

    Give each button a role, and optionally a label (blank uses the role's own name, so renaming the role renames the button), an emoji, and a color. Leave the button list empty for a plain announcement.

    ⚠️
    Roles with mod powers can't go on a button

    Any role carrying Administrator, Manage Server / Roles / Channels / Messages / Webhooks, Kick, Ban, Timeout, or Mention Everyone is refused, as is any role above Dungeon Keeper in the role list. Unlike Role Menus there's no override — an announcement is public and stays clickable forever, so this one is a hard no. The check runs again on every click, so if a role picks up mod permissions later, the old button stops handing it out on its own. Members who already have the role can always still remove it.

    26 QA Tracker

    A volunteer testing crew for the bot itself. Each new change lands as a QA card in the testing channel — an embed carrying the test checklist with Pass / Fail / Blocked buttons. Members holding the configured QA-crew role click a verdict (Fail requires a "what went wrong" note, which lands in the card's thread); the first Pass turns the card green as verified. Your first verdict on a test pays economy currency instantly, up to a daily cap — re-clicking updates your verdict without re-paying. Until an admin configures the QA-crew role, the buttons are admins-only. A card that's stayed verified (green) for 10 minutes is automatically deleted from the channel to keep the queue tidy — the verdicts and payouts stay on record, just the message goes.

    ℹ️
    Admin oversight on the web dashboard

    The Dev-section QA Tracker panel (admin-only) has the status board with every test and its verdicts, a top-testers scoreboard, and the config knobs: QA-crew role, channel, reward amount, daily cap, and enable toggle. A bogus verdict can be voided there — the paid coins are clawed back and the card's status recomputes. Archiving retires a card.
